# 宠爱到家 - 宝塔面板部署指南

## 📋 环境要求

- 宝塔面板最新版本
- PHP 8.0+
- MySQL 5.7+ 或 8.0
- Nginx 或 Apache Web服务器
- Composer（用于安装PHP依赖）

## 🚀 部署步骤

### 第一步：上传项目文件

1. 登录宝塔面板
2. 点击左侧菜单 **文件**
3. 进入 `/www/wwwroot/` 目录
4. 点击 **上传**，将整个 `chongai-tp8` 文件夹上传到此目录

或者使用FTP/SFTP工具上传：

```bash
cd /www/wwwroot/
rz -y  # 如果有zip文件
unzip chongai-tp8.zip
```

### 第二步：创建数据库

1. 登录宝塔面板
2. 点击左侧菜单 **数据库**
3. 点击 **添加数据库**
4. 填写信息：
   - 数据库名：`chongai`
   - 用户名：`chongai`（或自定义）
   - 密码：自动生成或自定义（**请记住密码**）
   - 选择：`utf8mb4` 编码
5. 点击 **提交**

### 第三步：导入SQL数据

1. 在数据库列表中，找到刚创建的数据库
2. 点击 **管理** 按钮（会打开phpMyAdmin）
3. 点击左侧数据库名称
4. 点击顶部菜单 **导入**
5. 点击 **选择文件**，找到并选择 `chongai-tp8/sql/chongai.sql`
6. 点击页面底部的 **执行** 按钮

### 第四步：安装PHP依赖

1. 打开宝塔终端（或者SSH连接到服务器）
2. 进入后端目录：

```bash
cd /www/wwwroot/chongai-tp8/backend
```

3. 安装依赖：

```bash
composer install
```

如果提示没有composer，运行：

```bash
curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer
composer install
```

### 第五步：配置后端

1. 编辑环境配置文件：

```bash
nano /www/wwwroot/chongai-tp8/backend/.env
```

2. 修改数据库连接信息：

```env
APP_DEBUG = false

DB_TYPE = mysql
DB_HOST = 127.0.0.1
DB_NAME = chongai
DB_USER = chongai
DB_PASS = your_password_here
DB_PORT = 3306
DB_CHARSET = utf8mb4
DB_PREFIX = chongai_
```

替换：
- `DB_USER`：你创建的数据库用户名
- `DB_PASS`：你设置的数据库密码

3. 保存文件（Ctrl+X，然后Y，回车）

### 第六步：设置目录权限

```bash
chmod -R 755 /www/wwwroot/chongai-tp8/backend/runtime
chmod -R 755 /www/wwwroot/chongai-tp8/backend/public
chown -R www:www /www/wwwroot/chongai-tp8/backend/runtime
chown -R www:www /www/wwwroot/chongai-tp8/backend/public
```

### 第七步：创建网站（后端API）

1. 点击宝塔左侧菜单 **网站**
2. 点击 **添加站点**
3. 填写信息：
   - 域名：`api.yourdomain.com`（填写你的域名，如果没有可以填IP）
   - 根目录：`/www/wwwroot/chongai-tp8/backend/public`
   - PHP版本：选择 `PHP-8.0` 或更高
4. 点击 **提交**

### 第八步：配置后端网站

1. 在网站列表中找到刚创建的网站
2. 点击 **设置**
3. 点击 **伪静态**，选择 `thinkphp`，点击 **保存**

如果列表中没有thinkphp，手动添加：

```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
        break;
    }
}
```

4. 点击 **配置文件**，在配置文件中添加（解决跨域问题）：

在 `server {` 块内的 `location / {` 之前添加：

```nginx
# 允许跨域
add_header 'Access-Control-Allow-Origin' '*' always;
add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE, OPTIONS' always;
add_header 'Access-Control-Allow-Headers' 'Content-Type, Authorization, X-Requested-With' always;

if ($request_method = 'OPTIONS') {
    return 204;
}
```

5. 保存并重载Nginx配置

### 第九步：创建前端网站

1. 再次点击 **添加站点**
2. 填写信息：
   - 域名：`www.yourdomain.com`（填写你的主域名）
   - 根目录：`/www/wwwroot/chongai-tp8/frontend`
   - 不需要PHP版本（纯静态）
3. 点击 **提交**

### 第十步：配置前端

1. 编辑前端API配置文件：

```bash
nano /www/wwwroot/chongai-tp8/frontend/js/api.js
```

2. 修改API地址为你的后端域名：

```javascript
var API_BASE_URL = 'https://api.yourdomain.com';
```

3. 保存文件

### 第十一步：配置SSL证书（推荐）

1. 在后端API网站设置中
2. 点击 **SSL**
3. 选择 **Let's Encrypt** 或上传已有证书
4. 开启 **强制HTTPS**
5. 保存并重载Nginx配置

### 第十二步：测试网站

#### 测试后端API

访问：`https://api.yourdomain.com/api/pets`

应该返回JSON数据，包含宠物列表

#### 测试前端网站

访问：`https://www.yourdomain.com/index.html`

应该能看到宠物领养网站首页

#### 测试管理员后台

访问：`https://api.yourdomain.com/admin.html`

使用以下账号登录：
- 用户名：`admin`
- 密码：`123456`

## ⚠️ 常见问题

### 1. 数据库连接失败

**错误信息**：`SQLSTATE[HY000] [2002] Connection refused`

**解决方案**：
- 检查 `.env` 文件中的数据库信息是否正确
- 检查MySQL服务是否运行：`systemctl status mysqld`
- 检查端口是否正确（默认3306）

### 2. 权限不足

**错误信息**：`Permission denied`

**解决方案**：

```bash
chmod -R 755 /www/wwwroot/chongai-tp8/backend/runtime
chmod -R 755 /www/wwwroot/chongai-tp8/backend/public
chown -R www:www /www/wwwroot/chongai-tp8/backend/runtime
chown -R www:www /www/wwwroot/chongai-tp8/backend/public
```

### 3. 跨域问题

**错误信息**：`Access-Control-Allow-Origin` 相关错误

**解决方案**：
- 确保在Nginx配置中添加了跨域头
- 确保后端中间件正确配置
- 重载Nginx配置

### 4. 404 Not Found

**错误信息**：`The page you visited does not exist`

**解决方案**：
- 检查Nginx伪静态规则是否配置
- 检查路由配置文件 `route/app.php` 是否存在
- 清理缓存

### 5. Composer内存不足

**错误信息**：`Killed` 或内存不足

**解决方案**：

```bash
php -d memory_limit=-1 /usr/local/bin/composer install
```

## 🔐 安全建议

1. **修改数据库密码**：将 `.env` 中的数据库密码改为强密码
2. **修改JWT密钥**：在 `ApiAuth.php` 和 `UserController.php` 中修改 `$secret` 值
3. **关闭调试模式**：将 `.env` 中的 `APP_DEBUG` 改为 `false`
4. **使用HTTPS**：务必配置SSL证书，启用HTTPS
5. **定期备份**：定期备份数据库和文件
